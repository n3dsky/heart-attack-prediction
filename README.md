# Heart Attack Risk Prediction API

Сервис для предсказания риска сердечного приступа на основе медицинских данных пациентов.  
Модель машинного обучения (CatBoost) анализирует множество факторов (возраст, холестерин, давление, образ жизни и др.) и выдаёт вероятность высокого риска.  
API позволяет выполнять как одиночные, так и пакетные предсказания, а также загружать данные в формате CSV.    
Для проверки качества модели ревьюером, тестовые предсказания лежат по пути `app/test_predictions.csv`    
Реализовано тестовое приложение `app/test.py` - для помощи запустить `python ./app/test.py --help`

## Возможности

- **Загрузка CSV** с данными нескольких пациентов и получение предсказаний для каждого.
- **Одиночное предсказание** через JSON-объект с признаками.
- **Пакетное предсказание** (список JSON-объектов).
- **Проверка загрузки модели** и доступности сервиса.
- **Статистика** по результатам: количество пациентов с высоким риском, процент, средняя вероятность.

## Установка и запуск

### Требования

- Python 3.8 или выше

### Установка зависимостей

```bash
pip install -r requirements.txt
```

*Содержимое requirements.txt:*

```
fastapi==0.104.1
uvicorn==0.24.0
pandas==2.1.3
numpy==1.24.3
scikit-learn==1.3.2
joblib==1.3.2
requests==2.31.0
python-multipart==0.0.6
pydantic==2.7.0
python-dotenv==1.0.0
```

### Запуск сервиса

```bash
python app.py
```

Сервер запустится на `http://0.0.0.0:8000`.  
Документация Swagger будет доступна по адресу `http://localhost:8000/docs`.

## Использование API

### Эндпоинты

#### Корневой эндпоинт

**GET /**  
Возвращает общую информацию о сервисе, список доступных эндпоинтов и статус загрузки модели.

#### Проверка состояния

**GET /health**  
Проверяет работоспособность сервиса и наличие загруженной модели.

#### Список признаков

**GET /features**  
Возвращает список всех признаков, ожидаемых моделью, а также общее количество признаков.

#### Предсказание из CSV

**POST /predict/csv**  
Загрузите CSV-файл с данными пациентов. Файл должен содержать все необходимые колонки (см. /features).  
Ответ содержит предсказания (0/1), вероятности и статистику.

#### Одиночное предсказание

**POST /predict/single**  
Отправьте JSON с признаками одного пациента согласно схеме `HealthMetrics`.  
Ответ содержит предсказание (0/1), вероятность и уровень риска (Низкий/Высокий).

#### Пакетное предсказание

**POST /predict/batch**  
Отправьте массив JSON-объектов (каждый соответствует `HealthMetrics`).  
Ответ аналогичен ответу для CSV.

### Примеры запросов

#### Одиночное предсказание

```bash
curl -X POST "http://localhost:8000/predict/single" \
  -H "Content-Type: application/json" \
  -d '{
    "Age": 55,
    "Cholesterol": 240,
    "Heart_rate": 78,
    "Diabetes": 1,
    "Family_History": 0,
    "Smoking": 1,
    "Obesity": 0,
    "Alcohol_Consumption": 0,
    "Exercise_Hours_Per_Week": 2.5,
    "Diet": 0,
    "Previous_Heart_Problems": 0,
    "Medication_Use": 0,
    "Stress_Level": 6,
    "Sedentary_Hours_Per_Day": 8,
    "Income": 45000,
    "BMI": 28.5,
    "Triglycerides": 150,
    "Physical_Activity_Days_Per_Week": 3,
    "Sleep_Hours_Per_Day": 7,
    "Blood_sugar": 110,
    "CK_MB": 3.5,
    "Troponin": 0.02,
    "Gender": "Male",
    "Systolic_blood_pressure": 130,
    "Diastolic_blood_pressure": 85
  }'
```

Ответ:

```json
{
  "status": "success",
  "prediction": 1,
  "probability": 0.78,
  "risk_level": "Высокий",
  "message": "Риск сердечного приступа: Высокий (вероятность: 78.00%)"
}
```

#### Загрузка CSV

```bash
curl -X POST "http://localhost:8000/predict/csv" \
  -F "file=@patients.csv"
```

*patients.csv* должен содержать те же признаки, что и в JSON (имена колонок могут быть с пробелами или подчёркиваниями — сервис автоматически сопоставит).

Ответ:

```json
{
  "status": "success",
  "predictions": [1, 0, 1],
  "probabilities": [0.82, 0.12, 0.67],
  "filename": "patients.csv",
  "statistics": {
    "total_patients": 3,
    "high_risk_count": 2,
    "low_risk_count": 1,
    "high_risk_percentage": 66.67,
    "avg_probability": 0.5367
  },
  "message": "Успешно обработано 3 пациентов"
}
```

## Структура проекта и описание классов

Проект построен по принципу разделения ответственности. Основные компоненты:

### `DataPreprocessor`

Отвечает за предобработку данных перед подачей в модель.

- **`expected_features`**: список всех признаков, ожидаемых моделью.
- **`categorical_features`**: признаки, которые должны быть целочисленными (категориальные).
- **`column_mapping`**: словарь для преобразования имён колонок из стиля Python (snake_case) в исходные имена, использованные при обучении.
- **`prediction_threshold`**: порог для отнесения к классу высокого риска (по умолчанию 0.29).
- **`validate_columns(df)`**: проверяет наличие всех обязательных колонок в DataFrame.
- **`preprocess(df)`**: выполняет преобразование пола в числовой формат, заполнение пропусков, приведение категориальных признаков к целому типу.
- **`prepare_single_record(data)`**: создаёт DataFrame из словаря с признаками одного пациента, переименовывает колонки и применяет предобработку.
- **`remove_target_column(df)`**: удаляет целевую переменную, если она присутствует в данных.

### `ModelPredictor`

Управляет загрузкой модели и выполнением предсказаний.

- **`model_path`**: путь к файлу модели (по умолчанию `catboost_model.cbm`).
- **`model`**: загруженный объект CatBoostClassifier.
- **`is_loaded`**: флаг успешной загрузки модели.
- **`load()`**: загружает модель из файла.
- **`predict(df, threshold)`**: возвращает предсказания (0/1) и вероятности для каждого экземпляра.

### `StatisticsCalculator`

Содержит статические методы для расчёта статистики по результатам пакетного предсказания.

- **`calculate_batch_statistics(predictions, probabilities)`**: возвращает словарь с общим количеством пациентов, числом высокорисковых, процентом и средней вероятностью.

### `PredictionService`

Основной сервис, координирующий предобработку, предсказание и статистику.

- **`predict_from_dataframe(df)`**: выполняет полный цикл предсказания для DataFrame.
- **`predict_single(metrics)`**: предсказание для одного пациента (объект `HealthMetrics`).
- **`predict_batch(df)`**: пакетное предсказание с возвратом объекта `BatchPredictionResult`.
- **`predict_from_csv(contents, filename)`**: читает CSV из байтов, создаёт DataFrame и вызывает `predict_batch`.

### `HeartAttackRiskAPI`

Собирает все компоненты вместе и регистрирует эндпоинты FastAPI.

- **`__init__(model_path)`**: инициализирует приложение FastAPI, компоненты и вызывает `_setup_routes()`.
- **`_setup_routes()`**: определяет все маршруты API.
- **`get_app()`**: возвращает экземпляр FastAPI для запуска.

### Модели данных

- **`HealthMetrics`**: Pydantic-модель, описывающая все входные признаки с валидацией (диапазоны, регулярные выражения).
- **`PredictionResult`**: dataclass для результата одиночного предсказания (prediction, probability, risk_level).
- **`BatchPredictionResult`**: dataclass для пакетного результата (списки предсказаний и вероятностей, статистика, общее количество).
- **`PredictionResponse`**: Pydantic-модель ответа API для пакетных запросов.
- **`RiskLevel`**: Enum с уровнями риска (Низкий/Высокий).

### Логирование

Настроено базовое логирование через модуль `logging`. Все ключевые события (загрузка модели, ошибки) фиксируются.
